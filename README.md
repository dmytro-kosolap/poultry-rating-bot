# 🐔 Рейтинг надійних постачальників — Telegram Bot

## Файли
- `bot.py` — основний код бота
- `database.py` — робота з базою даних (SQLite)
- `requirements.txt` — залежності

---

## Розгортання на Railway

### Крок 1 — Завантажте файли
Завантажте всі три файли (`bot.py`, `database.py`, `requirements.txt`) в репозиторій GitHub або напряму в Railway через Dashboard.

### Крок 2 — Змінні середовища (Environment Variables)
У Railway Dashboard → ваш проєкт → вкладка **Variables**, додайте:

| Змінна           | Значення                          | Обов'язково |
|------------------|-----------------------------------|-------------|
| `BOT_TOKEN`      | токен від @BotFather               | ✅          |
| `ADMIN_IDS`      | ваш Telegram ID (наприклад `123456789`) | ✅    |
| `MIN_DAYS_TO_VOTE` | `180` (або `1` для тестування)  | ні          |
| `MIN_DAYS_TO_BE_RATED` | `180`                       | ні          |
| `DB_PATH`        | `/data/rating.db`                 | рекомендовано |

> Ваш Telegram ID можна дізнатись у бота @userinfobot

### Крок 3 — Persistent Volume (важливо!)
SQLite зберігає дані у файл. Щоб дані не зникали після рестарту:
1. Railway Dashboard → ваш сервіс → **Volumes**
2. Додайте Volume, Mount Path: `/data`
3. Встановіть `DB_PATH=/data/rating.db` у Variables

### Крок 4 — Запуск
Railway автоматично виявить `requirements.txt` і встановить залежності.
Start команда: `python bot.py`

---

## Налаштування бота в групі

1. Додайте бота в групу як **адміністратора**
2. Видайте права: читати повідомлення, бачити нових учасників
3. Надішліть `/start` в групу — бот готовий!

---

## Команди

| Команда | Хто може | Опис |
|---------|----------|------|
| `/top` | всі | Топ-10 постачальників |
| `/check @username` | всі | Профіль і відгуки продавця |
| `/rate @username` | учасники 6+ міс. | Залишити оцінку |
| `/mystatus` | всі | Мій статус і статистика |
| `/admin_freeze @username` | адміни | Заморозити акаунт |
| `/admin_unfreeze @username` | адміни | Розморозити акаунт |

---

## Захист від маніпуляцій

- Мінімум 6 місяців в групі для голосування
- Мінімум 6 місяців для отримання оцінок  
- Один голос від одного учасника (можна оновити)
- Адмін може заморозити підозрілий акаунт
- Всі оцінки логуються з датами

---

## Тестування

Для тестування встановіть `MIN_DAYS_TO_VOTE=1` і `MIN_DAYS_TO_BE_RATED=1` у Variables — тоді обмеження по часу не заважатимуть.
